The Problem

The Raw Data is Unusable

SEC EDGAR publishes thousands of filings — but they're locked in formats no AI or analyst can query directly.

📄

Unstructured HTML

10-K filings are massive HTML documents. Item 1 (Business), Item 1A (Risk Factors), and Item 7 (MD&A) are buried inside raw markup with no clean boundaries.

<div class="WordSection1"><p class="MsoBodyText">...thousands of lines...</p></div>
🧮

XBRL Concept Chaos

The same metric — "Revenue" — can appear under 5+ different XBRL concept names depending on the company's filing software.

Revenues
SalesRevenueNet
RevenueFromContractWithCustomer...
→ all mean "revenue"
🔁

No Idempotency

Re-running ingestion jobs on the SEC API without careful design causes duplicate records, phantom data, and broken analytics. Every re-run must be safe.

❌ Naïve Append
= duplicates
✅ Delta MERGE
= idempotent
Data source: SEC EDGAR — /api/xbrl/companyfacts/{cik}.json
The Solution

Medallion Architecture

A 5-layer production pipeline — every stage adds structure, quality, and value.

Data Quality by Design

Idempotent MERGE

Every pipeline layer uses Delta MERGE — safe to re-run at any time.

SHA-256 Deduplication

Deterministic IDs on statements and chunks prevent phantom rows.

Change Data Feed

delta.enableChangeDataFeed on all Bronze tables for downstream CDC.

🗺️

XBRL Normalisation

A single TARGET_CONCEPT_MAP canonically maps 30+ raw us-gaap concepts to 11 clean metric names. The single source of truth.
